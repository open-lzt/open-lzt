# lzt-flows — официальный реестр модулей

Готовые флоу для flow-движка [auto-lzt](https://github.com/open-lzt/auto-lzt). Установка — одной командой в боте:

```
/modules          — что есть
/import bump-daily — поставить
```

## Что такое модуль

Каталог в `modules/` из двух файлов:

- `module.yaml` — имя, версия, автор, описание;
- `flow.json` — сам граф.

**Модуль — это данные, не код.** Ни `.py`, ни скриптов: CI отклоняет любой файл, кроме этих двух
и `README.md`. Иначе реестр незаметно превратился бы в канал распространения кода, который никто
не проверяет как канал распространения кода.

## Как добавить свой

1. Один раз добавьте себя в `authors.yml` — отдельным PR.
2. Отдельным PR добавьте `modules/<имя>/`. `author` в манифесте должен совпадать с автором PR.
3. CI прогонит `lzt-flow-validate` — **тот же самый валидатор**, который потом запустит бэкенд при
   установке. Прошло CI — значит, установится; не прошло — вы увидите причину в PR, а не потом.
4. После мержа `index.json` пересобирается автоматически. Руками его не трогайте — PR с ним
   отклоняется.

Модуль отклоняется, если он: ссылается на неизвестный узел, использует узел с возможностью
`reflective` (произвольный вызов API маркета), не компилируется настоящим компилятором, или несёт
исполняемые файлы.

## Честно про доверие

`sha256` в `index.json` — это **целостность передачи**: он доказывает, что файл не изменился между
этим репозиторием и вашей установкой. Это **не подпись** и он ничего не говорит о том, можно ли
доверять автору.

Единственное, что защищает вас от вредного модуля, — то, что мейнтейнер прочитал его перед мержем.
Читайте `flow.json` сами: это обычный JSON, и в нём видно всё, что модуль сделает.
